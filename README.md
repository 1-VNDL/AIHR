# README: Мульти-агентная генерация презентации
## «Будущее рынка труда. ИИ в HR»

> **Аудитория:** студенты 3-го курса, направление «Менеджмент»
> **Объём:** 30–35 слайдов
> **Язык:** русский; английские аббревиатуры допустимы при наличии пояснения при первом упоминании
> **Данные:** приоритет — 2025, 2026 гг. При использовании данных необходимо указать источники (вкл. названия статей, исследований, опросов и прочего, из которых эти данные были взяты)

## 1. Архитектура мульти-агентной системы

### 1.1. Схема взаимодействия

```
                        ┌─────────────────────┐
                        │    ORCHESTRATOR      │
                        │  (координация и      │
                        │   сборка output)     │
                        └────────┬────────────┘
                                 │ параллельный запуск
          ┌──────────┬───────────┼───────────┬──────────┐
          ▼          ▼           ▼           ▼          ▼
      [Agent A]  [Agent B]  [Agent C]  [Agent D]  [Agent E]
      Разделы    Раздел 3   Раздел 4   Раздел 5   Разделы
       1–2                                          6–7
          │          │           │           │          │
          └──────────┴───────────┼───────────┴──────────┘
                                 │ output всех агентов
                                 ▼
                        ┌─────────────────────┐
                        │     SUPERVISOR       │
                        │  (критическая        │
                        │   проверка)          │
                        └─────────────────────┘
                                 │ апрув / возврат на доработку
                                 ▼
                        ┌─────────────────────┐
                        │  ФИНАЛЬНАЯ СБОРКА    │
                        │  (Orchestrator)      │
                        └─────────────────────┘
```

### 1.2. Роли агентов

#### 🎯 Orchestrator (Оркестратор)

**Ответственность:**
- Инициализация параллельных задач для Agent A–E
- Передача единого контекста (общая рамка, tone of voice, ограничения) всем агентам до старта
- Сбор и стыковка разделов в единую структуру после завершения параллельной работы
- Управление зависимостями: Supervisor получает output только после завершения **всех** контентных агентов
- Финальная сборка презентации после прохождения супервайзерской проверки
- Разрешение конфликтов терминологии и нумерации между разделами

**Входные данные:** настоящий README
**Выходные данные:** 1) задания агентам A–E; 2) структурированный markdown; 3) финальная сборка презентации в pptx (через вызов view /mnt/skills/public/pptx/SKILL.md)

#### 📘 Agent A — «Контекст и рынок труда»

**Разделы:** 1 (Вводная часть) + 2 (Рынок труда: тренды)
**Слайдов:** 6–7

**Зона ответственности:**
- Слайд-повестка и навигация по презентации
- Глобальные и российские тренды на рынке труда 2024–2026
- Статистика занятости и автоматизации
- Контекстная рамка для всей презентации (задаёт «почему это важно»)

**Ограничения:**
- Все цифры — с указанием источника и года
- Исключить академические формулировки; ориентир — разговор с 20–25-летней студенческой аудиторией, без "детского заигрывания" и мемов

#### 🤖 Agent B — «ИИ-инструменты в HR»

**Раздел:** 3
**Слайдов:** 6–7

**Зона ответственности:**
- Карта ИИ-инструментов по HR-функциям: подбор, адаптация, обучение, оценка, аналитика
- Примеры реальных платформ и их применение (с кейсами)
- До-/после автоматизации: как меняется работа HR-специалиста
- Обязательный кейс: ≥1 российская компания, ≥1 международная

**Ограничения:**
- Технические термины — с пояснением при первом упоминании
- Не менее 1 схемы / диаграммы (описание слайда с визуалом)
- Избегать vendor-специфичного контента без нейтрального контекста

---

#### 💼 Agent C — «Трансформация профессий»

**Раздел:** 4
**Слайдов:** 5–6

**Зона ответственности:**
- Классификация профессий: под угрозой / трансформируются / появляются новые
- Навыки будущего: what skills survive vs. become obsolete
- Данные о конкретных специальностях в управлении и HR
- Интерактивная пауза: «Оцени свою специальность» (формат дискуссии / опроса)

**Ограничения:**
- Данные — не старше 2023 г. (приоритет 2024–2026)
- Избегать апокалиптических формулировок: фокус на трансформации, не на замещении
- Обязательна балансировка: риски + возможности

---

#### ⚖️ Agent D — «Этика, риски и человеческий фактор»

**Раздел:** 5
**Слайдов:** 5–6

**Зона ответственности:**
- Bias (предвзятость) в алгоритмах подбора и оценки персонала
- Приватность данных сотрудников при использовании ИИ
- Регуляторная повестка: EU AI Act, российское регулирование 2024–2025
- Роль человека в системах с ИИ («human-in-the-loop»)
- Дискуссионный слайд: этическая дилемма для аудитории

**Ограничения:**
- Не уходить в правовые детали — уровень «что нужно знать менеджеру»
- Обязательна практическая рекомендация по каждому риску
- Тон: взвешенный, не алармистский

---

#### 🎓 Agent E — «Стратегия студента + Заключение»

**Разделы:** 6 (Как подготовиться) + 7 (Итоги)
**Слайдов:** 5–6

**Зона ответственности:**
- Практическая дорожная карта: что делать студенту прямо сейчас
- Навыки и компетенции, повышающие ценность в эпоху ИИ
- Карьерные траектории на стыке менеджмента и HR-tech
- Итоговый слайд с ключевыми выводами (не более 5 тезисов)
- Завершающая дискуссия / Q&A-блок

**Ограничения:**
- Конкретные, реалистичные рекомендации — без «развивайте soft skills» без расшифровки
- Финальные выводы должны перекликаться с тезисами из Раздела 1 (смысловое кольцо)
- Тон: мотивирующий, но реалистичный

---

#### 🔍 Supervisor (Супервайзер)

**Тип работы:** последовательная проверка **после** завершения всех контентных агентов
**Входные данные:** объединённый output Agent A–E

**Чеклист проверки:**

| Критерий | Описание |
|---|---|
| **Фактология** | Все статистические данные имеют источник и год; данные не противоречат друг другу между разделами |
| **Терминология** | Одинаковые понятия названы одинаково во всех разделах; ИИ-термины пояснены при первом упоминании |
| **MECE** | Разделы не дублируют смысловые блоки друг друга; вместе покрывают тему |
| **Action titles** | Каждый слайд имеет заголовок-вывод, а не заголовок-тему |
| **Логика повествования** | Переходы между разделами связаны; финал перекликается с введением |
| **Доступность** | Отсутствуют необъяснённые технические термины; язык соответствует уровню студентов-менеджеров |
| **Интерактивность** | В презентации ≥3 интерактивных паузы (дискуссия / опрос) |
| **Визуалы** | Каждый раздел содержит ≥1 описание схемы / диаграммы / инфографики |
| **Плотность контента** | Контент каждого слайда заполняет 70–80% площади (Arial 14, формат 16:9) |

**Выходные данные:** отчёт с замечаниями по каждому агенту (Pass / Rework + комментарий) → передаётся Orchestrator для финальной сборки или возврата агентам

## 3. Единые требования для всех контентных агентов (A–E)

### 3.1. Обязательные стандарты слайда

```
Каждый слайд должен содержать:
  ✅ Action title — формулировка-вывод, не тема
  ✅ Не более 5–7 смысловых единиц на слайде
  ✅ Визуальный элемент ИЛИ структурированный список (не сплошной текст)
  ✅ Источник для любой цифры (год + организация)
  ✅ Контент (без заголовка и списка источников) заполняет 70–80% площади слайда
     (шрифт Arial 14, формат 16:9)

Запрещено:
  ❌ Заголовки-темы («Анализ данных за 2025 год»)
  ❌ Сплошной текст без структуры
  ❌ Данные без указания источника и года
  ❌ Необъяснённые технические термины
```

### 3.2. Формат передачи output агентом

Каждый агент передаёт Orchestrator структурированный markdown в формате:

```markdown
## [Номер раздела]: [Название]
### Слайд [X.Y]: [Action title]
- Тип: [статистика | схема | кейс | дискуссия | таблица]
- Содержание: [описание контента слайда]
- Визуал: [описание предполагаемой схемы/диаграммы]
- Источники: [список]
```

### 3.3. Требования к данным

- Приоритет источников: WEF, McKinsey, Gartner, hh.ru, Росстат, SuperJob, BCG, Deloitte
- Для российского контекста — обязательно наличие локальных данных (hh.ru, SuperJob, ЦБ РФ, Росстат)
- Прогнозы на 2026+ маркировать как прогнозы с указанием организации-автора

## 4. Workflow: последовательность работы

```
Шаг 1: Orchestrator читает README и формирует задания для Agent A–E
        (единый контекст: тема, аудитория, тон, ограничения)

Шаг 2: Agent A, B, C, D, E работают ПАРАЛЛЕЛЬНО
        Каждый генерирует контент своих разделов в формате из §3.2

Шаг 3: Orchestrator собирает output всех агентов
        Проверяет полноту (все разделы получены)

Шаг 4: Supervisor получает объединённый output
        Проверяет по чеклисту из §1.2
        Выдаёт отчёт: Pass ✅ / Rework ⚠️ с комментарием по каждому агенту

Шаг 5a (Pass): Orchestrator финализирует и собирает презентацию
Шаг 5b (Rework): Orchestrator направляет замечания Supervisor
                  конкретным агентам на доработку → повторный шаг 4

Шаг 6: Финальная сборка и экспорт
```

## 5. Глоссарий ключевых терминов
*(для согласованного использования всеми агентами)*

| Термин | Как использовать в презентации |
|---|---|
| ИИ (Искусственный интеллект) | Расшифровывать при первом упоминании в каждом разделе |
| HR (Human Resources) | «HR» допустимо без расшифровки — термин известен аудитории |
| L&D (Learning & Development) | При первом упоминании: «обучение и развитие персонала (L&D)» |
